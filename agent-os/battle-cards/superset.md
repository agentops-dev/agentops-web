# Battle Card: Superset
*Last updated: 2026-03-30*

## At a Glance
- **What they do:** Terminal/IDE for running 10+ parallel AI coding agents across isolated git worktrees; tool-agnostic
- **Founded:** 2026 | **Funding:** $0 (bootstrapped) | **Team:** 3 founders
- **Price:** Free (ELv2 open source)
- **Model:** Free; users supply own API keys
- **Best for:** Power developers who want maximum agent parallelism and full CLI/terminal control

## Their Strengths (be honest)
- Strongest GitHub traction among coding agent tools (8.3K stars, Feb 2026 launch) [Data]
- #2 Product Hunt Day launch with 553 upvotes [Data]
- Most tool-agnostic: any CLI agent works (Claude Code, OpenCode, Codex, Cursor, Gemini, Aider) [Data]
- Highest parallelism claim (10+ agents) [Data]
- Terminal-native approach appeals to power developer audience
- "Trusted by" Microsoft, OpenAI, Netflix, DoorDash, Salesforce, Google, Vercel on landing page [Data — though nature of "trust" unclear]

## Their Weaknesses (your openings)
- **Very new company** — founded 2026, no funding, 3 people [Data]
- **ELv2 license is not truly open source** — commercial restrictions may cause community friction [Opinion]
- **No organizational layer** — parallel execution only; no hierarchy, no CHARTER, no context
- **No cross-session memory** — agents start fresh each time
- **Terminal-native may limit adoption** — mainstream developers may prefer GUI
- **No revenue/sustainability plan visible** — like Conductor, free product is a time-limited strategy

## How to Win Against Them
- **When they say "Superset lets me run 10 agents at once":** "Great for execution throughput. Agent OS lets you organize what those 10 agents should be doing, ensure they don't repeat each other's work, and carry context forward from session to session."
- **When they say "Superset is free and works with everything":** "Agent OS's core is also free and open source — and it adds organizational structure that Superset doesn't have."
- **Lead with:** "Superset maximizes how many agents run. Agent OS maximizes how well your agents are organized. For a solo dev on a small project, maybe you only need parallelism. For any team using agents seriously, you need both."

## When They Win Over You
- **Terminal-only power developers who hate GUIs** — Superset's terminal aesthetic is exactly what they want
- **Maximum parallelism (10+ agents)** — if raw scale is the need, Superset delivers
- **Teams needing cross-platform parallelism on Linux** — better Linux support than Conductor

## Their Customers' Top Complaint
ELv2 license restrictions may become a concern as the project scales [Opinion — not yet a documented complaint]

## Key Vulnerability
No funding + 3 founders + newly founded = very high abandonment risk. Building developer tools at this pace without backing is a marathon that most teams don't complete. If Superset stalls or is abandoned, their 8.3K star GitHub becomes an acquisition target.

## Churn Signals
- If ELv2 restricts commercial usage and community responds negatively
- As teams grow and need organizational management beyond execution

## Watch For
Superset is early enough that their roadmap could go in many directions. If they add CHARTER-like features or organizational hierarchy, they become more directly competitive. Given no funding, probability of this happening quickly is low.
